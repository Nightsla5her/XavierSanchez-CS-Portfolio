# XavierSanchez-CS-Portfolio

The applications I developed for this course were designed to help students register for courses while following specific business rules, such as preventing duplicate registrations and limiting total credit hours. The Console version focused on core logic and user input, while the WPF version expanded the project into a graphical interface that allowed users to interact with buttons, lists, and text fields. Both versions demonstrate my ability to work with existing code, debug errors, and implement new features based on software requirements.

I did particularly well in organizing the code and keeping the logic clear and readable. I also made sure the WPF interface was user‑friendly by providing clear messages and preventing invalid actions. Comparing the two versions helped me understand how UI design affects user experience. The Console app required only text output, while the WPF app needed visual components like combo boxes, labels, and error messages to guide the user.

My debugging approach was systematic: I tested frequently, used Visual Studio’s error list, and fixed issues one at a time. These strategies helped me catch mistakes early and will be useful in future projects. I also had to be innovative when converting the program to use arrays and when adapting the logic for the WPF interface. Managing state between UI elements and enforcing business rules required careful planning.

Overall, this project strengthened my understanding of C#, debugging, UI design, and working with existing codebases. It also gave me a strong artifact to include in my portfolio that demonstrates both technical skill and attention to user‑centered design.
